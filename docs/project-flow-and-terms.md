# 멀티모달 서빙 실험 문서 안내

이 문서는 `multimodal-serving-lab`의 상위 안내 페이지다. 용어 설명과 프로젝트 흐름을 한 파일에 모두
넣으면 문서가 길어지고 목적이 섞이므로, 아래처럼 두 문서로 나눠 읽는다.

| 문서 | 목적 | 먼저 읽을 상황 |
|---|---|---|
| [`project-flow.md`](project-flow.md) | 전체 실험 흐름, 서브 프로젝트별 로직 흐름, 실험 판단 기준 | 저장소 구조와 코드 흐름을 이해하고 싶을 때 |
| [`glossary.md`](glossary.md) | 멀티모달/서빙/최적화 용어 사전 | RTF, TTFT, dynamic batching 같은 용어를 확인하고 싶을 때 |
| [`experiments.md`](experiments.md) | 실제 실험 수치와 관찰 기록 | 결과 수치와 결론 근거를 확인하고 싶을 때 |

## 추천 읽기 순서

1. [`project-flow.md`](project-flow.md)로 전체 구조와 데이터 흐름을 먼저 본다.
2. 모르는 용어가 나오면 [`glossary.md`](glossary.md)를 확인한다.
3. 실측 수치와 세부 결론은 [`experiments.md`](experiments.md)에서 확인한다.
4. 실제 실행 방법은 각 서브 디렉터리의 `README.md`를 따른다.

## 문서 분리 기준

| 분리한 영역 | 이유 |
|---|---|
| 프로젝트 흐름 | 서브 프로젝트별 입력, 처리, 출력, 핵심 파일을 빠르게 비교하기 위함 |
| 용어 사전 | 개념 설명을 독립적으로 검색하고 재사용하기 위함 |
| 실험 기록 | 수치와 관찰 로그가 길어져 상위 설명과 섞이지 않게 하기 위함 |

더 세분화하지 않은 이유는 이 저장소가 아직 여러 독립 제품이 아니라 하나의 실험 아카이브이기 때문이다.
현재 수준에서는 `흐름`, `용어`, `실험 기록` 세 축으로 나누는 것이 가장 읽기 쉽다.
